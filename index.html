Redesign U.S Department of Education Website

Project Overview

The goal of this individual project is to understand the pain points for users navigating the U.S Department of Education, and to develop a responsive website design.

Role - UX/UI Designer

Jessica Liu

introduction

Challenges Statement

The US Department of Education’s website is provided by the US government. It provides educational information to support students and educators, such as student loans, grants, educational law, or education data..etc. My challenge in this project is to discover the pain points when users navigated the Ed.gov site.

Goal

I will do user research and analysis UI navigation.

I will improve the site navigation base on user needs.

I will redesign the layout into a responsive website.

User Testing Insights:

  • Didn’t realize “Student Loan” was a clickable header
  • Use more images to help the user navigate to sections
  • Under Navigation bar, the subtitles is not clickable, but image is clickable.
  • Need a notification, when the user is going to another website.
  • Users have diffculit time to find what they needs. Need better organize, and less context.
  • Have to take a time to learn in the main page, because too much information.
  • The priority arrangement of search result.

How might we statement?

How might we improve the site so that information is easy to find, and support student and educator effeciently?

  • Redesign the navigation flow - the user used a search bar a lot, when they tried to look for their needs. The menu system and categories are not easy enough to understand. I discovered that some of the words were too similar that confused the users. The layout of content needs to be easy to read and add more pictures and space.
  • Redesign the website design - It needs a responsive design, especially the increase of numbers for using the phone.

Conclusion

The Department of Education’s website has a lot of designs that need to be improved. The site’s user experience is not friendly as shown in the testing results. When I conducted testing with users, most of the users had trouble finishing tasks successfully. Testings’ results gave me ideas on what needs to be improved on this site. After I started to redesign, I still wanted to keep some of the nice features and colors from the original design. I choose the same colors-gray blue and light green for color palette. I added more pictures in the homepage, because it’s text-heavy to the user, even to me. The website is not mobile-first design, or I think they didn’t have a mobile version. In my redesign, I decided to make a mobile-first design. I sketched the mobile version first, then started to rethink how those features showed in the desktop version. I found out that the user really relies on the search bar so I made search bars easy to find. A lot of links are directed to outside’s websites without a notification. The user always has a hard time going back to the original website. I wish I could make a notification that the user can click the link after choosing to “Leave site”, then open a new window in Figma, but I think Figma won't be able to hyperlink in the prototyping. In the future, I will definitely continue my redesign. There’s still more to do, such as make a filler for student loans and grants when students want to search…etc.
